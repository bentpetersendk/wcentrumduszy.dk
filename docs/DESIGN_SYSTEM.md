# Design System

## Foundations

The system should be quiet, premium, and humane. It should rely on spacing, typography, real photography, and restrained interaction rather than decorative effects.

## Colours

Use the brand palette defined in `BRAND_GUIDE.md`. Tokens:

- `color.background`
- `color.surface`
- `color.text`
- `color.textMuted`
- `color.accent.sage`
- `color.accent.clay`
- `color.accent.mist`
- `color.border`
- `color.error`
- `color.success`

## Typography

Scale:

- Display: 56-72px desktop, 40-48px mobile.
- H1: 44-56px desktop, 34-40px mobile.
- H2: 32-40px desktop, 28-32px mobile.
- H3: 24-28px.
- Body: 17-18px.
- Small: 14-15px.

Line height should be generous. Letter spacing should remain normal.

## Grid

- 12-column desktop grid.
- 8-column tablet grid.
- 4-column mobile grid.
- Max content width: 1200px.
- Reading width: 720px.

## Spacing

Spacing tokens: 4, 8, 12, 16, 24, 32, 48, 64, 80, 120.

## Buttons

Primary buttons are calm and confident. Secondary buttons are quiet. Button labels should be practical: Contact Joanna, View workshops, Read more, Join newsletter.

## Forms

Forms should use large fields, clear labels, optional helper text, visible validation, and calm confirmation states.

## Cards

Cards are for repeated items only. Page sections should not be floating card stacks. Card radius max: 8px.

## Icons

Use minimal line icons for utility. Avoid symbolic spiritual icon systems unless derived from the approved logo.

## Motion

Motion tokens:

- Fast: 150ms.
- Standard: 250ms.
- Slow: 450ms.
- Easing: soft ease-out.

## Responsive Behaviour

Mobile layouts should prioritize reading, trust, and clear CTAs. Avoid cramming multiple columns. Use stable aspect ratios for image cards and media.

## Accessibility

- WCAG 2.2 AA.
- Keyboard-first testing.
- Focus states on all controls.
- Reduced motion support.
- Text alternatives for images.
- Form labels always visible.
