# UI Reference

## Navigation

Desktop navigation is a quiet horizontal header with logo left, essential links centered or right, language selector, and one primary contact CTA. It should be sticky only after scroll and should reduce height slightly when fixed.

Mobile navigation uses a full-height drawer with large tap targets, language selector, and contact CTA at the bottom.

## Footer

The footer is a calm closing room. It includes logo, short brand statement, navigation, newsletter signup, contact details, social links, and legal links. Use a slightly darker warm paper or mist background.

## Buttons

Primary buttons: muted clay or deep charcoal fill, soft text contrast, 8px radius, no heavy shadow.

Secondary buttons: subtle border, transparent background.

Tertiary buttons: text link with underline or arrow on hover.

States: default, hover, focus, active, disabled, loading.

## Forms

Forms use visible labels, generous inputs, warm paper surfaces, clear validation, and calm success messages. Never hide labels inside placeholders only.

## Cards

Cards are used for repeated content only: workshops, articles, courses, meditations, testimonials, gallery images. Radius max 8px. Use subtle borders instead of heavy shadows.

## Article Pages

Article pages are editorial:

- narrow reading column
- large title
- quiet metadata
- hero image
- generous paragraph rhythm
- related content at the end

## Workshop Pages

Workshop pages balance emotion and practical details:

- strong title and short description
- date, time, location, language, format, capacity
- image or atmosphere block
- agenda
- who it is for
- FAQ
- booking/contact CTA

## Gallery

Gallery uses calm masonry or disciplined grids. Images open in a lightbox with caption, alt text, and usage context. Avoid dramatic overlays.

## Booking Flow

Future booking flow should be simple:

1. Select workshop or course.
2. Review practical details.
3. Enter contact details.
4. Confirm request or payment.
5. See calm success page.

## CMS

CMS UI must feel like editing a calm social page:

- dashboard cards for quick actions
- plain labels
- preview button always visible
- autosave status
- publish confirmation
- no database or deployment terms

## Language Selector

Use a compact text selector: `PL`, `DA`, `EN`. Desktop can show inline. Mobile should place it inside the drawer. Always indicate current language.

## Search

Search should be quiet and content-focused. Use a single input with filters for content type only when needed. Results should show title, type, excerpt, and language.

## Mobile Navigation

Use a menu icon with clear accessible label. Drawer should slide or fade gently. Links should be full-width rows with 44px minimum height.

## 404 Page

Use a calm message, one portrait or subtle texture, and links to Home, Articles, Workshops, and Contact.

## Loading States

Use warm neutral skeletons matching the final layout. Avoid loud spinners.

## Success Pages

Success pages should feel like exhale moments: short confirmation, what happens next, and one gentle next action.

## Error Pages

Error pages should be factual and reassuring. Avoid technical detail. Provide a recovery link and contact option.
