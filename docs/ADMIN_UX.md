# Admin UX

## Design Goal

The admin should feel calm, obvious, and forgiving. Joanna should always know where she is, what is published, and what will happen when she clicks a button.

## Workshop Editor

Joanna edits title, description, date, location, capacity, price, booking status, practical details, FAQ, gallery, and CTA. Required fields appear at the top until complete.

## Course Editor

Courses use a module-and-lesson structure with drag-and-drop ordering. Lessons can contain text, audio, video, downloads, and related meditations.

## Gallery Manager

## SEO Editor

SEO should be practical:

- Page title.
- Search description.
- Social image.
- URL slug.
- Preview for Google and social sharing.
- Warnings for missing metadata.

## Navigation Editor

Navigation should use simple drag-and-drop ordering with visible labels per language. Joanna can hide a page from navigation without unpublishing it.

## Contact Messages

Messages should show sender, email, reason, date, status, and quick actions: mark replied, archive, copy email, open mail client.

## Images

Image screens should never mention storage buckets. They should say Library, Upload, Crop, Caption, Alt text, and Where used.
